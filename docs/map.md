# 公式マップに載せる（任意・opt-in）

ptp mesh は、希望するノードだけを **[meshmap.net](https://meshmap.net)** に掲載できます（公式ブローカーへの一方向ミラー）。**既定では誰も・どのグループも公開されません。**

## 載せる条件

掲載条件は「**OK to MQTT が ON** かつ **位置情報が流れている**」の 2 つ。ただし参加のかたちで意味が変わります。

**GW として参加している場合（MQTT 接続）** — OK to MQTT（アプリ → 無線設定 → **LoRa**）が、そのまま**マップに載せるかどうかの opt-in スイッチ**です。OFF でもブリッジの利用には影響しません（自分の発信は自分で uplink されるため）。載せたい場合は ON にして、位置情報を流してください（スマホ proxy ならアプリの位置共有、固定ノードなら固定座標）。

**GW を利用して参加している場合（[MQTT 接続なし](connect.md#gw-mqtt)）** — 参加自体に OK to MQTT ON が必須なので、条件 1 は常に成立しています。つまり**位置情報を流すかどうかがそのまま載る/載らないの選択**です。

!!! warning "GW 利用の参加者へ"
    位置情報を broadcast していると（スマホ連携の位置共有含む）、**自動的に世界地図への掲載対象になります**。載りたくない場合は位置共有をオフにしてください。

どちらの場合もサーバーへの申請などは不要で、条件が揃えば自動的に載ります。

!!! tip "位置の精度設定（推奨）"
    **チャンネル設定 → 位置 → 「正確な位置」は最大以外に**（10〜16bit ≒ ±350m〜数km に丸める）。
    精密位置のままだと公式サーバー側で破棄され、逆に地図に出ません。プライバシー的にも精密 OFF 推奨です。

## ⚠ 公開される情報

掲載すると以下が**全世界に公開**されます（私的なグループでも同様）:

- ノード名（long name / short name）
- 位置（上記の精度設定で丸めた座標）
- グループ名（root topic の文字列、例 `ptp/commons`）

メッセージ本文は一切ミラーされません（位置情報とノード情報のみ）。

## OK to MQTT ってそもそも何？

自分の発信を「他人のゲートウェイ経由でも MQTT に中継してよいか」の許可ビットです。

- **ON**：近隣の GW もあなたのパケットを中継します（＝カバレッジマップに載る／到達性が見える）。commons 参加なら基本 ON 推奨。
- **OFF**：他人の GW 経由の中継と、公式マップへの掲載を止めます。※あなた自身がゲートウェイ／スマホ proxy で繋いでいれば、自分の分の自己報告は行われます（OFF が止めるのは“他人による代理中継”だけ）。
- 混同注意：`ignore_mqtt` は別物（MQTT から降りてきたパケットを無視するダウンリンク側の設定）。通常は触らず OFF のままで。
