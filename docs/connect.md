# 参加方法

参加のかたちは 2 つ。**近くの GW の電波を借りる**か、**自分のノードを GW にする**か。

## GW を利用する（MQTT 接続不要）

自分のノードが MQTT に繋がっていなくても、**電波の届く範囲に ptp mesh の GW があれば**参加できます。あなたの RF 発信を近くの GW が拾ってブリッジに中継し、**他拠点の GW から RF 再放射**されます（逆方向も同様）。

| 設定項目 | 値 |
|---|---|
| LoRa → **OK to MQTT** | **ON**（既定 OFF のままだと GW があなたのパケットを中継できず、ブリッジに一切乗りません） |
| プライマリチャンネルの PSK | 既定（`AQ==`）のまま |
| preset / slot | **LongFast / slot12**（日本標準＝Meshtastic の既定設定そのまま。当ネットワークの全 GW がこれに統一されています） |

MQTT 設定は一切不要。GW の電波圏内に入るだけです。ただし到達確認の手段が少ないので、常用するなら GW 化がおすすめ。

なお、この形態で**位置情報を流していると自動的に[公式マップ](map.md)の掲載対象になります**（載りたくなければ位置共有をオフに）。

## GW として設定する（MQTT 接続）

ノードを MQTT サーバーに接続し、自分も中継点になる方法です。ノード自身の WiFi でも、スマホアプリの proxy 経由でも構いません。

| 設定項目 | 値 |
|---|---|
| MQTT 有効化 | ON |
| サーバー（Address） | `160.16.104.222:1883` |
| ユーザー名 / パスワード（推奨） | `friend` / `kcu8nxvtae72r4`（全部小文字と数字。紛らわしい文字なし） |
| ユーザー名 / パスワード（旧・引き続き有効） | `guest` / `lboAwDEzV1Hvh33O`（⚠先頭は小文字のエル、末尾は大文字のオー。手打ち注意） |
| 暗号化（Encryption） | **ON** |
| TLS | **OFF**（⚠ ON にすると繋がりません） |
| クライアント経由で接続（Proxy to Client） | スマホ proxy = **ON** ／ ノード自身が WiFi 直結 = **OFF**（逆だと無言で不通） |
| ルートトピック（Root topic） | `ptp/commons`（みんなのグループ）／ `ptp/<好きな名前>`（自分のグループ） |
| チャンネル → プライマリ → アップリンク / ダウンリンク | **両方 ON**（MQTT 設定とは別項目。忘れると無言で不通） |

preset は**自由**です（ブリッジが差を吸収）。commons に参加するときはプライマリ ch を既定 PSK のままに。

!!! tip "自分のグループを作りたい人へ"
    ルートトピックを `ptp/<好きな名前>` に変えるだけ。サーバーへの登録は不要です。私的にするなら名前付き ch＋独自 PSK も追加してください。

## ⚠ 注意（実験運用です）

- **TLS=OFF＝平文**。ユーザー名/パスワードは暗号化されずに流れます。身内向けの実験サーバーという前提で。
- **friend は全員共通アカウント**。荒らし対策等でパスワードを更新することがあります（その際は再周知します）。
- **commons は公開グループ**：誰でも読み書きできます。また **root 文字列は接続者からは見えます**（“無名性”は接続情報を持たない外部にしか効きません）。隠したい会話は私的グループ＋独自 PSK で。

## 繋がらない時

1. TLS が ON になっていないか
2. Proxy to Client の向き（スマホ proxy=ON ／ WiFi 直結=OFF）
3. チャンネル側の アップリンク／ダウンリンク が ON か（MQTT 設定とは別項目）
4. まず 20〜30 秒待つ（リブート直後は繋がりません）
